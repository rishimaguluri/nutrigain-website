External APIs

API Integrations

🍽️

Nutrislice API

Primary source for live OSU dining menus and nutrition data. Provides item-level calorie, macro, and allergen data for all campus dining locations.

PRIMARY SOURCE
🥦

USDA FoodData Central

Fallback source for macro and micronutrient data when items are not recognized by Nutrislice. Covers 300,000+ food items.

FALLBACK SOURCE
👁️

OpenAI Vision API

Converts a food photo into likely items ranked by confidence score. Powers the AI Meal Scanner feature — core Sprint 1 feature.

AI SCANNER
💬

GPT-4o-mini

Supports short natural-language dining questions such as "What gluten-free options are available tonight?" Fast and cost-efficient inference.

AI CHATBOT
🗺️

Leaflet Maps

Provides a no-cost, open-source way to support location-aware dining hall lookups. Renders the campus map with real-time busyness overlays.

MAPS / LOCATION
🎓

University Student Records

Via university partnership (FERPA-compliant). Pulls dietary restrictions and meal plan swipe balances on first login for zero manual setup — works with any university's student information system.

🗄️ Database Schema (Core Tables)
-- Users table CREATE TABLE users ( id UUID PRIMARY KEY, osu_id VARCHAR(10) UNIQUE NOT NULL, dietary TEXT[], -- ['vegan','gluten_free'] cal_goal INTEGER DEFAULT 2000, created_at TIMESTAMP ); -- Meal logs table CREATE TABLE meal_logs ( id UUID PRIMARY KEY, user_id UUID REFERENCES users(id), logged_at TIMESTAMP, calories INTEGER, protein_g DECIMAL, carbs_g DECIMAL, fat_g DECIMAL, items JSONB );

Core Functionality

Primary Functions

F1
Centralized Nutrition Access
Display calorie, macro, and allergen data for every campus dining hall menu item within 1 tap, sourced from the university's dining system.
F2
Real-Time Crowding Visibility
Color-coded hall occupancy status updated within ±2 minutes of actual sensor change, with 99% uptime SLA.
F3
Reduced Decision Fatigue
AI chatbot reduces decision fatigue by checking options across all campus locations at once using natural language.
F4
AI Meal Scanning & Logging
Photo-to-nutrition pipeline using OpenAI Vision, auto-logging to tracker. Zero manual data entry required.
F5
Dietary Profile Auto-Sync
Restrictions loaded from university student records with zero manual entry at first login. FERPA-compliant data access.
F6
Schedule-Aware Alerts
Proactive push notifications tied to class schedule and menu dietary compatibility, powered by Firebase Cloud Messaging.
F7
Habit Tracking Dashboard
Weekly nutrition summary, streak tracking, and personalized meal recommendations. Top 3 suggestions matched to goals.
F8
Swipe Balance Management
Track meal plan swipe balance and flag when weekly usage is running high. Integrates with any university campus card system.

Quality Assurance

Unit Testing & Verification Plan

🧪 Unit Testing Strategy
  • pytest for all Python backend functions — API endpoints, data transformations, and nutrition calculation logic
  • Test dining data scraper outputs against known campus menu items (calorie within ±5% tolerance)
  • Test AI scanner confidence scores on 20 standard campus meal photos — target ≥80% item match rate
  • Test chatbot responses on 50 pre-defined dining queries with expected output validation
  • Redis cache TTL tests — verify hall crowding data never exceeds 2-minute staleness
  • Database integrity tests — verify meal log entries persist correctly across sessions
